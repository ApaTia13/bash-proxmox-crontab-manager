# Proxmox Crontab Manager
Удобный интерактивный менеджер на Bash для управления задачами cron, управляющими виртуальными машинами и контейнерами Proxmox. Скрипт автоматически использует обёртку для безопасного выполнения команд, проверяет состояние ВМ/контейнеров перед выполнением действий, логирует результаты и может отправлять уведомления об ошибках в Telegram.

Проект представлен в двух версиях:
- [v1.0.0](https://github.com/ApaTia13/bash-proxmox-crontab-manager/releases/tag/1.0.0) – стабильная, проверенная на системах с большим количеством VM/LXC.
- [v1.1.1](https://github.com/ApaTia13/bash-proxmox-crontab-manager/releases/tag/1.1.1) – улучшенная версия с расширенными возможностями и повышенной безопасностью (прошла поверхностное тестирование, возможны баги).

## Автоматическая установка

Для быстрого развёртывания без ручной настройки пользователя, sudo-прав и загрузки скрипта воспользуйтесь [**автоматическим установщиком**](https://github.com/ApaTia13/proxmox-crontab-manager-installer). Он создаст пользователя `proxmox-scheduler`, настроит окружение, скачает выбранную версию менеджера и подготовит всё к работе.

## Логирование и мониторинг
- 📝 **Два типа логов**:
  - `actions_ГГГГ-ММ-ДД.log` – действия пользователя (добавление, редактирование, удаление).
  - `execution_ГГГГ-ММ-ДД.log` – результаты выполнения каждой задачи cron (команда, код возврата, вывод). Сюда же попадают сообщения о пропуске действий (если ВМ уже в нужном состоянии).
- 🗓️ **Ротация логов и бэкапов** – настраиваемый срок хранения (по умолчанию 30 дней), старые файлы удаляются автоматически. Можно настроить через меню (пункт 7).
- 📍 **Идентификация сервера** – в каждом логе и уведомлении указывается имя хоста и локальный IP-адрес, что позволяет различать события на нескольких серверах Proxmox.
- 🔔 **Уведомления об ошибках** – при ненулевом коде возврата задачи в Telegram отправляется сообщение с командой, описанием, кодом и первыми 500 символами вывода. HTML-сущности экранируются для корректного отображения.

## Безопасность и надёжность
| Механизм | Описание |
|----------|----------|
| **umask 027** | Для всех создаваемых файлов и директорий устанавливаются безопасные права (владелец — полный доступ, группа — чтение/исполнение, остальные — нет). |
| **Проверка конфигурационного файла** | При загрузке `~/.proxcron.conf` проверяется наличие потенциально опасных символов (`;`, `` ` ``, `$`, `()`, `{}`). При обнаружении загрузка отменяется. |
| **Экранирование HTML в Telegram** | Сообщения проходят через функцию экранирования `&`, `<`, `>`, `"` для предотвращения некорректного отображения. |
| Резервное копирование crontab | Перед любым изменением создаётся копия текущего crontab в `~/.cron.backups/`. |
| Запись через временный файл | Изменения сначала сохраняются во временный файл (создаётся через `mktemp`), затем атомарно заменяют crontab. |
| Восстановление при ошибке | Если запись не удалась, автоматически восстанавливается последний бэкап (если он не пуст и содержит данные). |
| Проверка синтаксиса | Базовая валидация cron-расписания (5 полей или специальные @-строки). |
| Поддержка специальных строк | `@reboot`, `@yearly`, `@annually`, `@monthly`, `@weekly`, `@daily`, `@hourly`. |
| Подтверждение действий | Запрос подтверждения перед добавлением, изменением или удалением задачи. |
| Безопасное хранение токена | Telegram-токен и chat_id хранятся в `~/.proxcron.conf` с правами **600** (устанавливается автоматически при настройке через меню). |
| Интеллектуальная обёртка | Перед выполнением команды проверяется текущее состояние VM/CT, чтобы избежать лишних ошибок (например, запуск уже запущенной ВМ). |
| Запрет запуска от root | Скрипт явно запрещает запуск от суперпользователя, требуя использования обычного пользователя с правами sudo. |
| Обработка ошибок парсинга | Если не удаётся извлечь статус ВМ/контейнера, команда всё равно выполняется, а ошибка логируется и (опционально) отправляется в Telegram. |
