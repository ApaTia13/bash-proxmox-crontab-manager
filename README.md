# Proxmox Crontab Manager

Удобный интерактивный менеджер на Bash для управления задачами cron, управляющими виртуальными машинами и контейнерами Proxmox. Скрипт автоматически использует обёртку для безопасного выполнения команд, проверяет состояние ВМ/контейнеров перед выполнением действий, логирует результаты и может отправлять уведомления об ошибках в Telegram.

## Возможности

- 📋 **Просмотр задач** – отображает все активные задачи из crontab, содержащие команды pct или qm, с нумерацией и человекочитаемым описанием расписания.
- ➕ **Добавление задачи** – интерактивный выбор ВМ/контейнера, действия (`start`, `stop`, `shutdown`, `reboot`, `suspend`, `resume`) и cron-расписания.
- ✏️ **Редактирование задачи** – изменение расписания (команда остаётся прежней).
- 🗑️ **Удаление задачи** – безопасное удаление с подтверждением.
- 📁 **Просмотр логов** -  последние 20 записей из логов действий пользователя и выполнения задач.
- 📊 **Статус cron-сервиса** – отображение активности, автозапуска и подробного состояния службы.
> **Добавление, редактирование и удаление - поддерживают множественный выбор.**

## Логирование и мониторинг

- 📝 **Два типа логов**:
  - `actions_ГГГГ-ММ-ДД.log` – действия пользователя (добавление, редактирование, удаление).
  - `execution_ГГГГ-ММ-ДД.log` – результаты выполнения каждой задачи cron (команда, код возврата, вывод). Сюда же попадают сообщения о пропуске действий (например, если ВМ уже запущена при попытке запуска).
- 🗓️ **Ротация логов и бэкапов** – логи и бэкапы хранятся 30 дней, старые удаляются автоматически.
- 📍 **Идентификация сервера** – в каждом логе и уведомлении указывается **имя хоста** и **локальный IP-адрес**, что позволяет различать события на нескольких серверах Proxmox.
- 🔔 **Уведомления об ошибках** – при ненулевом коде возврата задачи в Telegram отправляется сообщение с командой, описанием, кодом и первыми 500 символами вывода (требуется настройка Telegram-бота).

## Безопасность и надёжность

| Механизм | Описание |
|----------|----------|
| Резервное копирование crontab | Перед любым изменением создаётся копия текущего crontab в `~/.cron.backups/`. |
| Запись через временный файл | Изменения сначала сохраняются во временный файл, затем атомарно заменяют crontab. |
| Восстановление при ошибке | Если запись не удалась, автоматически восстанавливается последний бэкап. |
| Проверка синтаксиса | Валидация полей времени (диапазоны, списки, шаги), наличие команды. |
| Поддержка специальных строк | `@reboot`, `@yearly`, `@annually`, `@monthly`, `@weekly`, `@daily`, `@hourly`. |
| Подтверждение действий | Запрос подтверждения перед добавлением, изменением или удалением задачи. |
| Безопасное хранение токена | Telegram-токен и chat_id хранятся в `~/.proxcron.conf` с правами **600**. |
| Интеллектуальная обёртка | Перед выполнением команды проверяется текущее состояние VM/CT, чтобы избежать лишних ошибок (например, запуск уже запущенной ВМ). |
| Запрет запуска от root | Скрипт явно запрещает запуск от суперпользователя, требуя использования обычного пользователя с правами sudo. |

## Требования

- Хост Proxmox (наличие команд `pct` и `qm`).
- Пользователь с правами на выполнение `pct` и `qm` через `sudo` без пароля (настройки в `/etc/sudoers.d/`).
- Bash 4+.
- `curl` для отправки уведомлений в Telegram (опционально).

## Установка
### Получите скрипт
Вариант А: Скачать из репозитория (рекомендуется)
```bash
# Создайте директорию для скрипта (например, /opt/proxmox-cron-manager)
sudo mkdir -p /opt/proxmox-cron-manager
cd /opt/proxmox-cron-manager

# Скачайте скрипт с помощью wget или curl
sudo wget -O proxcron.sh https://raw.githubusercontent.com/ApaTia13/bash-proxmox-crontab-manager/refs/heads/main/proxcron.sh
# Или: sudo curl -o proxcron.sh https://raw.githubusercontent.com/ApaTia13/bash-proxmox-crontab-manager/refs/heads/main/proxcron.sh
```
Вариант Б: Скопировать с другого сервера через scp
На исходном сервере выполните
```bash
scp /путь/до/proxcron.sh пользователь@IP-целевого-сервера:/opt/proxmox-cron-manager/
```
### Установите `curl`
Если вы планируете использовать Telegram-уведомления
```bash
sudo apt update && sudo apt install curl -y
```
> Если curl не установлен, скрипт будет работать, но уведомления об ошибках отправляться не будут (в логах появится предупреждение)
### Настройте скрипт и конфигурацию
Перейдите в директорию со скриптом и сделайте его исполняемым
```bash
cd /opt/proxmox-cron-manager
sudo chmod +x proxcron.sh
```
Создайте конфигурационный файл для Telegram (если нужны уведомления)
```bash
nano ~/.proxcron.conf
```
Запишите в него
```bash
TELEGRAM_TOKEN="ваш_токен_бота"
TELEGRAM_CHAT_ID="ваш_chat_id"
```
Установите правильные права доступа
```bash
chmod 600 ~/.proxcron.conf
```
### Настройте sudo-права для команд proxmox-scheduler
Убедитесь, что ваш пользователь может выполнять `pct` и `qm` через `sudo` без пароля
```bash
sudo visudo -f /etc/sudoers.d/proxmox-scheduler
```
Добавьте строку (замените username)
```bash
proxmox-scheduler ALL=(ALL) NOPASSWD: /usr/sbin/qm start *, /usr/sbin/qm stop *, /usr/sbin/qm shutdown *, /usr/sbin/qm status *, /usr/sbin/qm list, /usr/sbin/qm reboot *, /usr/sbin/qm suspend *, /usr/sbin/qm resume *

proxmox-scheduler ALL=(ALL) NOPASSWD: /usr/sbin/pct start *, /usr/sbin/pct stop *, /usr/sbin/pct shutdown *, /usr/sbin/pct status *, /usr/sbin/pct list, /usr/sbin/pct reboot *, /usr/sbin/pct suspend *, /usr/sbin/pct resume *
```
### Запустите скрипт
```bash
/opt/proxmox-cron-manager/proxcron.sh
```
При первом запуске автоматически будут созданы
- обёртка `proxcron_wrapper.sh` в той же директории
- папка `logs` для хранения логов
- папка `~/.cron.backups` для резервных копий crontab
### (Опционально) Добавьте alias для удобства
```bash
echo "alias proxcron='/opt/proxmox-cron-manager/proxcron.sh'" >> ~/.bash_aliases
source ~/.bash_aliases
```
Теперь можно запускать менеджер командой `proxcron`
